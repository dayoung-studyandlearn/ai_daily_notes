## 오늘의 개념: RAG(검색 증강 생성)이란
RAG는 AI가 답변을 만들기 전에 먼저 관련 자료를 검색해서 참고한 뒤, 그 내용을 바탕으로 답을 생성하는 방식이에요. 마치 시험을 볼 때 책이나 자료를 참고할 수 있는 '오픈북 시험'과 비슷하다고 생각하면 쉽습니다. 이렇게 하면 AI가 원래 학습하지 못했던 최신 정보나 특정 문서의 내용도 답변에 반영할 수 있어요. 또한 근거 자료를 함께 제시하기 때문에 AI가 사실과 다른 내용을 지어내는 현상(환각)을 줄이는 데도 도움이 됩니다. 그래서 최신 뉴스 답변, 회사 내부 문서 기반 챗봇 등에서 RAG가 많이 활용되고 있어요.

## 오늘의 용어: RAG (검색 증강 생성, Retrieval-Augmented Generation)
RAG는 AI가 답변을 만들기 전에 먼저 외부 문서나 데이터베이스에서 관련 정보를 검색한 뒤, 그 내용을 참고해서 답을 생성하는 방식입니다. 쉽게 말하면, AI가 자기 기억(학습된 지식)에만 의존하지 않고 마치 '책을 찾아보고 답하는 것'처럼 최신 자료나 근거 자료를 찾아서 더 정확하고 신뢰할 수 있는 답변을 만들어내는 기술입니다. 이를 통해 AI가 모르는 최신 정보나 특정 문서 내용도 잘 답할 수 있게 됩니다.

## 예제 코드
```python
# RAG(검색 증강 생성) 개념을 이해하기 위한 간단한 예제

# 1. 검색 대상이 될 간단한 문서 저장소(지식 베이스)
documents = [
    "RAG는 검색과 생성을 결합한 AI 기법이다.",
    "파이썬은 배우기 쉬운 프로그래밍 언어이다.",
    "딥러닝은 인공신경망을 기반으로 한다.",
]

# 2. 사용자의 질문
query = "RAG가 뭐야?"

# 3. 검색(Retrieval) 단계: 질문과 겹치는 단어가 가장 많은 문서를 찾음
def retrieve(query, documents):
    query_words = set(query.replace("?", "").split())  # 질문을 단어 집합으로 변환
    best_doc = ""
    best_score = 0
    for doc in documents:
        doc_words = set(doc.split())  # 문서도 단어 집합으로 변환
        score = len(query_words & doc_words)  # 겹치는 단어 수 계산
        if score > best_score:
            best_score = score
            best_doc = doc
    return best_doc

# 4. 관련 문서 검색 실행
retrieved_doc = retrieve(query, documents)

# 5. 생성(Generation) 단계: 검색된 문서를 참고해 답변을 만듦 (실제로는 LLM이 수행)
def generate_answer(query, context):
    return f"질문: {query}\n참고 문서: {context}\n답변: {context}"

# 6. 최종 답변 출력
answer = generate_answer(query, retrieved_doc)
print(answer)
```
RAG(검색 증강 생성)의 핵심 개념을 아주 단순화하여 보여주는 예제입니다. 사용자의 질문과 가장 관련 있는 문서를 간단한 키워드 매칭으로 '검색'한 뒤, 그 문서를 참고 자료로 활용해 '생성'된 답변처럼 출력합니다. 실제 RAG는 임베딩 기반 유사도 검색과 LLM을 사용하지만, 이 예제는 그 흐름을 이해하기 위한 최소 버전입니다.

## 실무 팁
RAG 구축 시 문서를 무작정 512~1000 토큰 단위로 잘라 임베딩하지 말고, 문단/제목 구조를 유지하는 '의미 단위 청킹(semantic chunking)'을 적용하세요. 예를 들어 LangChain의 RecursiveCharacterTextSplitter를 쓸 때 chunk_size=500, chunk_overlap=50 정도로 설정하고, 마크다운 헤더나 문단 구분자를 우선 분리 기준으로 삼으면 검색 정확도가 크게 올라갑니다. 또한 검색 결과를 LLM에 넘기기 전 'Rerank' 단계(예: Cohere Rerank나 cross-encoder 모델)를 추가하면, 단순 벡터 유사도 검색보다 실제 답변 품질이 눈에 띄게 개선됩니다.

## 오늘의 퀴즈
Q. RAG(검색 증강 생성)를 사용하는 AI 모델이, RAG 없이 사전 학습된 지식만 사용하는 모델과 비교했을 때 '최신 정보'나 '특정 문서 기반 질문'에 더 정확하게 답할 수 있는 이유는 무엇인가요? RAG의 동작 과정을 바탕으로 설명해보세요.
A. RAG는 사용자의 질문이 들어오면 먼저 외부 데이터베이스나 문서 저장소에서 질문과 관련된 정보를 검색(Retrieval)하고, 이 검색된 정보를 프롬프트에 함께 포함시켜 LLM이 답변을 생성(Generation)하도록 합니다. 즉, 모델이 학습 시점의 고정된 지식에만 의존하지 않고 실시간으로 관련 외부 정보를 참고하기 때문에, 학습 데이터에 없던 최신 정보나 특정 문서의 세부 내용에 대해서도 근거 있는 정확한 답변을 생성할 수 있는 것입니다.
