## 오늘의 개념: 임베딩(Embedding)이란 무엇인가
임베딩은 단어, 문장, 이미지 같은 데이터를 컴퓨터가 이해할 수 있도록 숫자로 이루어진 벡터(숫자 목록)로 바꾸는 방법입니다. 예를 들어 '고양이'와 '강아지'는 의미가 비슷하기 때문에, 임베딩을 하면 두 단어의 벡터 값이 서로 가까운 위치에 놓이게 됩니다. 반대로 '고양이'와 '자동차'처럼 의미가 다른 단어들은 벡터 공간에서 멀리 떨어지게 됩니다. 이렇게 의미를 숫자 공간의 '거리'와 '방향'으로 표현하면, 컴퓨터가 단어나 문장의 유사도를 계산하거나 추천, 검색, 번역 같은 작업을 훨씬 효과적으로 수행할 수 있습니다. 즉, 임베딩은 사람이 이해하는 의미를 컴퓨터가 계산할 수 있는 숫자 형태로 '번역'해주는 다리 역할을 한다고 생각하면 됩니다.

## 오늘의 용어: 임베딩 (Embedding)
임베딩은 단어, 문장, 이미지 같은 데이터를 컴퓨터가 이해할 수 있도록 숫자로 이루어진 벡터(좌표 값)로 바꾸는 방법이에요. 이렇게 바꾸면 의미가 비슷한 것들은 벡터 공간에서 가까운 위치에 놓이게 되어, 컴퓨터가 '고양이'와 '강아지'가 '자동차'보다 더 비슷하다는 것을 계산으로 알 수 있게 됩니다. AI가 텍스트나 이미지의 의미를 파악하고 비교하는 데 핵심적으로 사용되는 기술이에요.

## 예제 코드
```python
import numpy as np

# 단어를 숫자 벡터(임베딩)로 표현한 예시 (실제로는 학습된 모델이 만듦)
word_embeddings = {
    "고양이": np.array([0.9, 0.1, 0.2]),
    "강아지": np.array([0.8, 0.2, 0.1]),
    "자동차": np.array([0.1, 0.9, 0.8]),
}

# 두 벡터 사이의 코사인 유사도를 계산하는 함수
def cosine_similarity(vec1, vec2):
    dot_product = np.dot(vec1, vec2)          # 내적 계산
    norm1 = np.linalg.norm(vec1)               # 벡터1의 크기
    norm2 = np.linalg.norm(vec2)               # 벡터2의 크기
    return dot_product / (norm1 * norm2)       # 코사인 유사도 공식

# 고양이 vs 강아지 (의미가 비슷하므로 유사도가 높을 것)
sim1 = cosine_similarity(word_embeddings["고양이"], word_embeddings["강아지"])
print(f"고양이-강아지 유사도: {sim1:.3f}")

# 고양이 vs 자동차 (의미가 다르므로 유사도가 낮을 것)
sim2 = cosine_similarity(word_embeddings["고양이"], word_embeddings["자동차"])
print(f"고양이-자동차 유사도: {sim2:.3f}")

```
임베딩(Embedding)이란 단어나 문장 같은 데이터를 숫자로 이루어진 벡터로 변환하는 것을 의미합니다. 이 예제는 간단한 단어-벡터 딕셔너리를 만들어 임베딩 개념을 보여주고, 두 단어 벡터 간의 코사인 유사도를 계산하여 의미적으로 얼마나 비슷한지 확인합니다. 실제로는 Word2Vec, GloVe, BERT 같은 모델이 훨씬 정교한 임베딩을 만들어냅니다.

## 실무 팁
임베딩은 텍스트/이미지 등을 의미를 담은 고정 길이 숫자 벡터로 변환한 것으로, 벡터 간 거리(코사인 유사도 등)가 가까울수록 의미적으로 유사하다는 것을 뜻합니다. 실무에서 검색/추천/RAG 시스템을 만들 때는 OpenAI의 text-embedding-3-small 같은 모델로 문서를 임베딩한 뒤 FAISS나 Pinecone 같은 벡터DB에 저장하고, 사용자의 쿼리를 같은 모델로 임베딩해 코사인 유사도 top-k를 뽑아 검색하면 키워드 매칭보다 훨씬 정확한 의미 기반 검색을 바로 구현할 수 있습니다. 단, 임베딩 모델은 반드시 문서 저장 시점과 쿼리 검색 시점에 동일한 모델을 사용해야 벡터 공간이 일치해 정확한 유사도 비교가 가능합니다.

## 오늘의 퀴즈
Q. 임베딩(Embedding)이란 무엇이며, 왜 컴퓨터가 텍스트나 이미지 같은 데이터를 그대로 이해하지 못해서 임베딩이 필요한지 설명해보세요. 또한 '의미가 비슷한 단어일수록 임베딩 공간에서 가깝게 위치한다'는 말은 무슨 뜻인가요?
A. 임베딩이란 텍스트, 이미지 등 사람이 이해하는 데이터를 컴퓨터가 계산할 수 있는 숫자 벡터(고정된 길이의 실수 배열)로 변환하는 것입니다. 컴퓨터는 단어나 이미지 자체의 '의미'를 직접 이해할 수 없기 때문에, 이를 수치화된 벡터로 표현해야 유사도 계산이나 학습, 검색 등을 수행할 수 있습니다. '의미가 비슷한 단어일수록 가깝게 위치한다'는 것은, 임베딩 벡터를 다차원 공간의 한 점으로 봤을 때 '왕'과 '여왕'처럼 의미적으로 관련된 단어들은 벡터 간 거리(또는 코사인 유사도)가 가깝게 학습되고, 관련 없는 단어들은 멀리 떨어지도록 벡터가 만들어진다는 뜻입니다.
