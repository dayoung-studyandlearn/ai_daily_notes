## 오늘의 개념: LLM(거대 언어 모델)이 뭔가요?
LLM은 'Large Language Model'의 줄임말로, 아주 많은 글(책, 웹사이트, 코드 등)을 미리 읽고 "이 문장 다음에 어떤 단어가 올 확률이 높은가"를 아주 잘 맞히도록 훈련된 프로그램입니다. Claude, ChatGPT 같은 서비스가 모두 LLM을 기반으로 만들어졌습니다. 핵심은 이겁니다 — LLM은 '생각해서' 답하는 게 아니라, 지금까지의 글을 보고 '다음에 나올 확률이 가장 높은 단어'를 계속 이어 붙이는 방식으로 문장을 만듭니다. 그런데 이 예측을 워낙 정교하게 잘하다 보니, 마치 이해하고 대답하는 것처럼 보이는 것입니다.

## 오늘의 용어: API (Application Programming Interface)
서로 다른 두 프로그램이 정해진 규칙으로 대화하는 방법입니다. 식당에 비유하면 이해가 쉽습니다: 손님(우리가 만드는 프로그램)이 주방(Claude)에 직접 들어가지 않고, 메뉴판(API)에 적힌 대로 주문만 하면 웨이터가 요청을 전달하고 완성된 음식을 가져다줍니다. 우리 프로젝트의 generators/concept.py 코드가 하는 일이 바로 이 '주문 넣기'입니다.

## 예제 코드
```python
from anthropic import Anthropic

# API 키로 '나는 이 프로젝트다'라고 인증하는 창구를 만듭니다
client = Anthropic()

response = client.messages.create(
    model="claude-sonnet-5",       # 어떤 AI 모델에게 물어볼지
    max_tokens=100,                   # 답변 길이를 얼마나 허용할지
    messages=[
        {"role": "user", "content": "안녕!"}  # 내가 보내는 메시지
    ],
)

print(response.content[0].text)  # AI가 준 답변 텍스트를 출력

```
Claude API에 메시지를 보내고 답을 받는 가장 기본적인 형태입니다. 실제로 실행하려면 API 키가 있어야 하지만, 지금은 '이런 모양이구나'만 보시면 됩니다. 한 줄씩 무엇을 하는지 주석으로 달아뒀습니다.

## 실무 팁
API 키는 절대로 코드 안에 직접 쓰지 말고 .env 파일에 따로 보관하세요. 이유는 간단합니다 — 키가 코드에 그대로 박혀 있으면, 나중에 그 코드를 GitHub 같은 곳에 올렸을 때 다른 사람이 그 키를 보고 마음대로 가져다 써서 내 돈이 청구될 수 있습니다. 실제로 이 프로젝트에서도 .env 파일은 .gitignore에 등록해서 git에는 절대 올라가지 않도록 만들어뒀습니다.

## 오늘의 퀴즈
Q. LLM이 문장을 만들어내는 방식을 가장 정확히 설명한 것은 무엇일까요?
A. 지금까지 나온 글을 보고, 다음에 올 확률이 가장 높은 단어(혹은 글자 조각)를 계속 예측해서 이어 붙이는 방식입니다. '이해하고 논리적으로 사고해서' 답하는 것이 아니라, 통계적으로 그럴듯한 다음 단어를 고르는 것에 가깝습니다.
