## 오늘의 개념: 벡터 데이터베이스 기초
벡터 데이터베이스는 텍스트, 이미지 같은 데이터를 숫자로 이루어진 '벡터'라는 형태로 바꿔서 저장하는 특별한 저장소예요. 여기서 벡터란 데이터의 의미나 특징을 여러 숫자의 조합으로 표현한 것이라고 생각하면 됩니다. 예를 들어 '강아지'와 '개'라는 단어는 뜻이 비슷하기 때문에 벡터로 바꿨을 때도 서로 가까운 위치에 놓이게 돼요. 벡터 데이터베이스는 이렇게 저장된 벡터들 중에서 내가 찾고자 하는 것과 '의미가 비슷한' 데이터를 빠르게 찾아주는 역할을 합니다. 그래서 AI 챗봇이 관련 정보를 검색하거나, 비슷한 이미지를 찾는 서비스 등에 널리 활용되고 있어요.

## 오늘의 용어: 벡터 데이터베이스 (Vector Database)
벡터 데이터베이스는 텍스트, 이미지 등의 데이터를 숫자로 이루어진 벡터(다차원 좌표값) 형태로 변환해 저장하고 검색하는 특별한 데이터베이스예요. 일반 데이터베이스가 '정확히 일치하는 값'을 찾는 데 특화되어 있다면, 벡터 데이터베이스는 '의미가 비슷한 것'을 빠르게 찾아주는 데 특화되어 있습니다. 예를 들어 '강아지'를 검색하면 '강아지'라는 단어가 없어도 의미가 비슷한 '반려견', '멍멍이' 같은 데이터를 찾아낼 수 있어서, 최근 AI 챗봇이나 추천 시스템에서 많이 활용돼요.

## 예제 코드
```python
import numpy as np

# 문장을 간단한 벡터(임베딩)로 표현 (실제로는 모델이 생성)
documents = ["나는 강아지를 좋아해", "나는 고양이를 좋아해", "오늘 날씨가 좋다"]
vectors = np.array([
    [0.9, 0.1, 0.0],  # "강아지" 문장 벡터
    [0.8, 0.2, 0.0],  # "고양이" 문장 벡터 (강아지와 유사)
    [0.0, 0.0, 1.0],  # "날씨" 문장 벡터 (전혀 다름)
])

# 검색할 질문(쿼리)도 벡터로 표현
query = np.array([0.85, 0.15, 0.0])

# 코사인 유사도 계산 함수 정의
def cosine_similarity(a, b):
    return np.dot(a, b) / (np.linalg.norm(a) * np.linalg.norm(b))

# 모든 문서 벡터와 쿼리 벡터 간 유사도 계산
similarities = [cosine_similarity(query, vec) for vec in vectors]

# 유사도가 가장 높은 문서의 인덱스 찾기
best_match_idx = int(np.argmax(similarities))

# 결과 출력
print("문서별 유사도:", [round(s, 3) for s in similarities])
print("가장 유사한 문서:", documents[best_match_idx])

```
벡터 데이터베이스의 핵심 개념인 '임베딩 생성'과 '유사도 검색'을 아주 간단하게 흉내낸 예제입니다. 실제 벡터DB(예: Chroma, Pinecone) 없이 numpy만으로 코사인 유사도를 계산해 가장 비슷한 문장을 찾는 원리를 보여줍니다. 입문자가 벡터 검색이 어떻게 동작하는지 감을 잡는 데 도움이 됩니다.

## 실무 팁
벡터 DB에 문서를 저장할 때 임베딩 벡터만 넣지 말고 원문 텍스트, 출처(source), 청크 위치(chunk_index) 등을 메타데이터로 함께 저장하세요. 예를 들어 Pinecone이나 Chroma에서 upsert할 때 metadata={"source": "doc1.pdf", "chunk": 3, "text": "원문내용"} 형태로 저장하면, 검색 결과를 사용자에게 보여주거나 RAG 답변에 출처를 표시할 때 별도의 DB 조회 없이 바로 활용할 수 있어 응답 속도와 디버깅 효율이 크게 향상됩니다.

## 오늘의 퀴즈
Q. 벡터 데이터베이스는 일반적인 관계형 데이터베이스와 달리 '유사도 검색(similarity search)'을 기반으로 데이터를 조회합니다. 텍스트나 이미지 같은 데이터를 벡터(임베딩)로 변환해서 저장하는 이유는 무엇이며, 이를 통해 어떤 방식으로 검색이 이루어지는지 설명해보세요.
A. 텍스트, 이미지 등의 데이터를 벡터(임베딩)로 변환하면 데이터의 '의미'나 '특징'을 고차원 공간상의 숫자 좌표로 표현할 수 있습니다. 이렇게 하면 정확히 일치하는 키워드가 아니더라도, 의미적으로 비슷한 데이터끼리는 벡터 공간에서 가까운 거리에 위치하게 됩니다. 벡터 데이터베이스는 검색어(쿼리)도 벡터로 변환한 뒤, 코사인 유사도나 유클리드 거리 같은 방법으로 저장된 벡터들과의 거리를 계산해 가장 가까운(유사한) 데이터를 찾아 반환합니다.
