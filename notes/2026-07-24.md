## 오늘의 개념: 파인튜닝 vs 프롬프트 엔지니어링
AI 모델을 원하는 목적에 맞게 활용하는 방법에는 크게 두 가지가 있어요. '파인튜닝'은 이미 학습된 AI 모델에게 특정 분야의 자료를 추가로 학습시켜서, 모델 자체의 지식과 성격을 바꾸는 방법이에요. 마치 요리를 이미 잘하는 사람에게 한식 요리를 특별히 더 훈련시켜서 한식 전문가로 만드는 것과 비슷해요. 반면 '프롬프트 엔지니어링'은 모델을 전혀 바꾸지 않고, 질문이나 요청(프롬프트)을 더 똑똑하고 구체적으로 작성해서 원하는 답변을 이끌어내는 방법이에요. 이는 같은 요리사에게 "그냥 요리해줘"가 아니라 "매콤하게, 15분 안에, 3인분으로 만들어줘"처럼 구체적으로 주문하는 것과 같아요. 즉, 파인튜닝은 모델 자체를 바꾸는 것이고, 프롬프트 엔지니어링은 모델은 그대로 두고 요청 방식만 바꾸는 것이라는 차이가 있습니다.

## 오늘의 용어: 파인튜닝(Fine-tuning) vs 프롬프트 엔지니어링(Prompt Engineering)
파인튜닝은 이미 학습된 AI 모델에게 특정 분야의 데이터를 추가로 학습시켜서 모델 자체의 지식과 성능을 조정하는 방법입니다. 반면 프롬프트 엔지니어링은 모델을 새로 학습시키지 않고, 질문이나 지시문(프롬프트)을 잘 설계해서 원하는 답변을 얻어내는 방법입니다. 쉽게 비유하면, 파인튜닝은 '학생을 추가로 교육시켜 실력을 키우는 것'이고, 프롬프트 엔지니어링은 '이미 아는 것이 많은 학생에게 질문을 잘해서 원하는 답을 이끌어내는 것'이라고 할 수 있습니다.

## 예제 코드
```python
# 간단한 가짜 모델: 입력에 특정 키워드가 있으면 답을 다르게 냄
def fake_model(prompt, weights=None):
    # weights가 있으면(파인튜닝된 모델) 항상 친절한 답변 스타일 사용
    if weights == "finetuned":
        return f"[친절한 답변] {prompt}에 대해 쉽게 설명해드릴게요!"
    # weights가 없으면 원본 모델, 프롬프트 내용에 따라서만 답변이 달라짐
    if "쉽게 설명해줘" in prompt:
        return f"[기본 모델] {prompt} -> 쉬운 버전으로 답변"
    return f"[기본 모델] {prompt} -> 기본 답변"

# 1. 프롬프트 엔지니어링 예시: 모델은 그대로, 입력 문장만 바꿔서 원하는 결과 유도
prompt1 = "AI란 무엇인가?"
prompt2 = "AI란 무엇인가? 쉽게 설명해줘"

print("=== 프롬프트 엔지니어링 ===")
print(fake_model(prompt1))  # 기본 답변
print(fake_model(prompt2))  # 프롬프트를 바꿔서 쉬운 답변 유도

# 2. 파인튜닝 예시: 모델 자체를 추가 학습시켜(weights 변경) 항상 원하는 스타일로 답변
print("\n=== 파인튜닝 ===")
print(fake_model(prompt1, weights="finetuned"))  # 프롬프트 그대로여도 모델이 바뀌어 친절한 답변
```
이 코드는 프롬프트 엔지니어링과 파인튜닝의 개념적 차이를 보여줍니다. 프롬프트 엔지니어링은 모델은 그대로 두고 입력(질문) 방식을 바꿔서 원하는 답을 유도하는 방식이고, 파인튜닝은 모델 자체(가중치)를 추가 학습시켜 원하는 답을 내도록 바꾸는 방식입니다. 예제는 실제 모델 학습 없이 두 방식의 흐름을 단순화하여 표현합니다.

## 실무 팁
모델의 답변 품질 문제가 생기면 먼저 프롬프트 엔지니어링(명확한 지시문, few-shot 예시 3~5개, 출력 포맷 명시)으로 해결을 시도하고, 그래도 특정 도메인 용어나 말투, 반복되는 형식이 안 잡힐 때만 파인튜닝을 고려하세요. 프롬프트 엔지니어링은 비용과 시간이 거의 들지 않고 즉시 수정 가능하지만, 파인튜닝은 데이터 준비·학습 비용이 크고 한 번 학습하면 수정이 번거롭기 때문에, 실무에서는 '프롬프트로 80% 해결 후 나머지 20%만 파인튜닝'하는 순서가 효율적입니다.

## 오늘의 퀴즈
Q. 동일한 사전학습 언어모델을 가지고 있을 때, '프롬프트 엔지니어링'과 '파인튜닝'은 모델의 지식이나 능력을 개선하는 방식에서 근본적으로 어떻게 다른가요? 각 방법이 모델의 내부 가중치에 미치는 영향을 중심으로 설명해보세요.
A. 프롬프트 엔지니어링은 모델의 내부 가중치(파라미터)를 전혀 변경하지 않고, 입력으로 주는 지시문이나 예시를 잘 설계해서 모델이 이미 가진 능력을 최대한 잘 이끌어내는 방법입니다. 반면 파인튜닝은 추가 데이터로 모델을 재학습시켜 실제 가중치 값을 업데이트함으로써 모델 자체의 행동이나 지식을 영구적으로 바꾸는 방법입니다. 따라서 프롬프트 엔지니어링은 빠르고 비용이 적게 들지만 매 요청마다 지시를 반복해야 하고, 파인튜닝은 시간과 비용이 더 들지만 한 번 학습시키면 모델이 그 특성을 지속적으로 유지합니다.
